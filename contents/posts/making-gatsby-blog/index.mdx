## 그래서, Gatsby가 뭔데? (개념 설명)

### 프레임워크란
Gatsby는 React를 기반으로 하는 웹 프레임워크입니다. 저번 편에서 '블로그를 쉽게 개발하기 위해 만들어둔 환경' 이라 소개했지만, 이건 너무 국소적인 설명이었네요.

**웹 프레임워크**는 웹사이트나 웹 애플리케이션을 만들 때 필요한 기본적인 구조와 기능들을 미리 제공하는 도구입니다. (기능 별로 제공하는게 아니라는 점에서 라이브러리와 구분됩니다.)
'Framework'(구조, 뼈대)라는 이름처럼, 집을 지을 때 기본 구조가 갖춰진 골조를 제공받는 것과 비슷하죠. 라이브러리는 이 골조 중 특정 기능을 하는 요소들로 비유할 수 있을 것 같습니다.
개발을 하는 과정에 있어 프레임워크는 크게 두가지 이점을 줍니다.
1. **기본 구조** : 사이트 제작에 필요한 기본적인 폴더 구조나 설정파일을 미리 세팅
2. **개발 편의성** : 반복적이거나, 빌드 과정에 필요한 코드들 제공
덕분에 우리는 바닥부터 모든 기능을 구현하는 대신, 이를 활용해서 빠르게 필요한 기능에 집중하면서 시간을 많이 절약할 수 있죠.

## Gatsby의 특징과 장점

Gatsby는 단순한 웹사이트 제작 도구를 넘어서는 현대적인 웹 프레임워크입니다. 주요 특징들을 살펴보면 다음과 같습니다:

### 1. Static Site Generator (정적 사이트 생성기)
Gatsby는 빌드 시점에 모든 페이지를 미리 생성합니다. 이렇게 생성된 정적 페이지들은 매우 빠른 로딩 속도를 제공하며, 검색엔진 최적화(SEO)에도 유리합니다.

### 2. React 기반 개발
React의 컴포넌트 기반 개발 방식을 그대로 활용할 수 있어, 재사용 가능한 UI 구성요소를 만들고 관리하기가 용이합니다.

### 3. GraphQL 데이터 레이어
Gatsby는 GraphQL을 통해 다양한 소스(마크다운 파일, CMS, API 등)에서 데이터를 가져와 통합적으로 관리할 수 있습니다. 이는 데이터 관리를 더욱 효율적으로 만들어줍니다.

### 4. 풍부한 플러그인 생태계
수많은 플러그인들이 제공되어 이미지 최적화, SEO 설정, 분석 도구 연동 등 다양한 기능을 쉽게 추가할 수 있습니다.

### 5. JAMstack 아키텍처
JavaScript, APIs, Markup(JAM)을 기반으로 하는 현대적인 웹 개발 아키텍처를 따르고 있어, 보안성과 확장성이 뛰어납니다.

## Gatsby는 어떻게 동작하나요?

1. 개발 단계에서는 React 컴포넌트로 웹사이트를 구성합니다.
2. GraphQL을 통해 필요한 데이터를 가져옵니다.
3. 빌드 시점에 모든 페이지를 정적 HTML로 생성합니다.
4. 사용자가 첫 페이지를 로드한 후에는 React의 클라이언트 사이드 라우팅을 통해 빠른 페이지 전환을 제공합니다.

이러한 특징들 덕분에 Gatsby는 블로그뿐만 아니라 기업 웹사이트, 전자상거래 플랫폼, 포트폴리오 등 다양한 용도의 웹사이트를 제작하는 데 적합합니다.
