`Gatsby` 프레임워크를 이용하기 위해서는 몇가지 사전 지식이 필요합니다. 먼저 Html, CSS 그리고 Java Script 언어입니다. (블로그 하나 호스팅해보려고 `ruby`와 `go`를 지나 무려 3번째 언어입니다)
물론 개념을 어느정도 이해해서 그때그때 필요한 내용을 찾아 해결해내는 정도면 충분하니, 걱정하지 말고 일단 시작하셔도 좋습니다!

## 그래서, `Gatsby`가 뭔데?
Gatsby는 React를 기반으로 하는 **웹 프레임워크**입니다. 저번 편에서 '블로그를 쉽게 개발하기 위해 만들어둔 환경' 이라 소개했지만, 이건 너무 국소적인 설명이었네요.

### 웹 프레임워크

**웹 프레임워크**는 웹사이트나 웹 애플리케이션을 만들 때 필요한 기본적인 구조와 기능들을 미리 제공하는 도구입니다. (기능 별로 제공하는게 아니라는 점에서 라이브러리와 구분됩니다.)
'Framework'(구조, 뼈대)라는 이름처럼, 집을 지을 때 기본 구조가 갖춰진 골조를 제공받는 것과 비슷하죠. 라이브러리는 이 골조 중 특정 기능을 하는 요소들로 비유할 수 있을 것 같습니다.

개발을 하는 과정에 있어 프레임워크는 크게 두가지 이점을 줍니다.

1. **기본 구조** : 사이트 제작에 필요한 기본적인 폴더 구조나 설정파일을 미리 세팅
2. **개발 편의성** : 반복적이거나, 빌드 과정에 필요한 코드들 제공
덕분에 우리는 바닥부터 모든 기능을 구현하는 대신, 이를 활용해서 빠르게 필요한 기능에 집중하면서 시간을 많이 절약할 수 있죠.

Gatsby 프레임워크는 기본적으로 자바스크립트 언어로 동작합니다. 이는 Gatsby의 구조 설계 특성상, 리액트의 컴포넌트를 활용한 기능 확장성에 중점을 두었기 때문으로 보입니다.
특히나 최근 프론트엔드에서 해당 기술이 거의 필수적인 것으로 보아, 사용자가 많고 필요한 정보를 쉽게 얻을 수 있다는 장점이 있습니다.

### Gatsby의 동작 과정

어떤 과정을 통해서 `gatsby`가 동작하는지 간단하게 살펴봅시다.

1. 개발 단계에서는 **React 컴포넌트**(자바스크립트 언어)로 웹사이트를 구성
2. **GraphQL**을 통해 필요한 데이터를 불러옴
3. 빌드 시점에 모든 페이지를 정적 HTML로 생성
4. 사용자가 첫 페이지를 로드하면 React의 클라이언트 사이드 라우팅을 통해 빠른 페이지 전환을 제공

이 과정에서 초보자인 우리가 이해해야 할 건,

1. "자바 스크립트로 웹사이트를 구성한다"
2. "GraphQL로 데이터를 불러온다"

이 두 가지만 짚으면 됩니다. 나머지 단계는 프레임워크가 알아서 해준다는 거니까요!

## Gatsby 튜토리얼 따라가기 (Part 1)

### 설치

Gatsby를 이용하기 위해서는 도구가 여러개 준비되어야 합니다.

1. Node.js (v18 - v21 사이)
2. Git
3. Gatsby command line interface (CLI) (v5 이상)
4. Code Editor (보통 VS code)

#### Node.js 설치
현재(2024년 12월) 기준으로 Node.js의 최신 버전은 v22 지만, Gatsby 공식 문서에서 지원하는 Node.js 버전은 v21까지므로, 되도록 다운그레이드하여 받아줍시다.

가장 쉬운 방법으로는 [`Node.js` 다운로드 페이지](https://nodejs.org/en/download) 에서 21 버전으로 맞춰서 다운받으시면 됩니다.

#### Git 설치
\[Mac]

보통 mac 에서는 XCode command line tool을 설치하면 자동으로 git이 깔려있으므로, 터미널에서 한번 확인해봅시다.

```shell
$ git --version
> git version 2.39.5 (Apple Git-154)
```
(사용자가 입력해야하는 건 $ 뒤에, 그 결과는 > 뒤에 나오도록 쓰겠습니다. 예를 들어, 여기서는 터미널에 `git --version`만 치시면 됩니다. )

\[Window]

윈도우 운영체제에서는 [홈페이지](https://gitforwindows.org/)에서 다운로드 해봅시다.

#### Gatsby CLI

위에서 `Node.js`를 잘 설치했다면 패키지 매니저인 npm(node package manager)도 설치되었을 것이므로, 바로 터미널에서 Gatsby CLI를 받아봅시다.

```shell
$ npm install -g gatsby-cli
```

잘 설치가 되면 `Gatsby`의 버전을 확인할 수 있습니다!

```shell
$ gatsby --version
```

#### Code Editor

공식 홈페이지에서는 [VS Code(홈페이지)](https://code.visualstudio.com/)를 권장하나, 저는 JetBrain 사의 [Webstorm(홈페이지)](https://www.jetbrains.com/webstorm/)을 이용하고 있습니다.
처음 에디터를 설치하신다면 대체로 VS Code를 설치하시길 권장합니다.
사용자가 많아 플러그인도 많고 문제가 있을 때 관련된 정보를 찾기도 쉽습니다.

이제 준비물은 끝입니다! 직접 웹사이트 하나 만들어 봅시다 :)
